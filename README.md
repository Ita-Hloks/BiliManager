# BiliManager

BiliManager 是一个基于 Manifest V3 的 Bilibili 浏览器扩展，使用 React、TypeScript、Tailwind CSS 和 Vite 构建。

## 功能

### 主题与背景：

支持浅色、深色和跟随系统主题，并可设置自定义背景（黑白遮罩会跟随系统主题）

<img src="./doc/dsp_bg.png" alt="主题与背景设置展示" width="80%">

### 搜索结果过滤：

按标题规则，UP 昵称（自定义正则表达式），互动率，模糊或降权视频。

<img src="./doc/dsp_fl.png" alt="搜索结果过滤展示" width="80%">

### 播放页个性化：

可选隐藏一些不重要的内容，并帮你记住相关推荐自动播放开关。
记录每日观看时长和最近观看视频。
（只要视频在播放，没有被最小化，都是可以加入统计的~）

数据同步：在扩展设置页导入或导出配置和观看历史，方便备份或迁移到另一台设备。

## 隐私说明

BiliManager 的设置与观看记录保存在当前浏览器配置文件中扩展专属的 `chrome.storage.local`。扩展不使用 `chrome.storage.sync`，不会通过浏览器账号同步这些数据，也不会将数据上传到开发者或第三方服务器。

- **设置配置**：保存功能开关、过滤规则、主题、定时器参数与位置，以及用户选择的自定义背景图片数据。
- **观看记录**：启用定时器后，保存播放日期、观看时长、页面标识、视频标题、视频 URL 和记录更新时间，用于统计观看时长和显示最近观看视频。
- **临时计时状态**：保存当前播放页的计时进度，便于页面刷新或短暂中断后继续统计；超过 12 小时的临时状态不会继续使用。
- **搜索页内容**：扩展会读取搜索结果中的标题、UP 主名称、播放量与弹幕量以执行本地过滤，但不会把这些搜索结果保存为观看历史。

观看历史最多保留 370 天，并限制为 5000 条索引记录；最近观看列表最多保留 100 个视频。用户可以在设置页导出或替换观看历史。卸载扩展时，浏览器通常会一并删除该扩展的本地存储

## 数据同步

打开扩展设置页的“数据管理”区域，可以使用“导入”和“导出”功能。

### 导出

- **设置配置**：功能开关、过滤规则、观看计时设置和主题等配置。
- **观看历史**：每日观看时长和最近观看视频记录。

### 导入

点击“导入”并选择之前导出的 JSON 文件即可恢复数据。导入全部数据或设置配置时，会保留当前设备上的自定义背景图片；导入观看历史时会替换当前已保存的观看历史。

导入前建议先导出当前数据作为备份。纯文本内容会按换行分隔，作为标题过滤规则导入。

## 开发与构建

```bash
npm install
npm run check
npm run build
```

构建完成后，在 Chrome 或 Edge 的扩展管理页面开启“开发者模式”，选择“加载已解压的扩展程序”，并载入生成的 `dist/` 目录。
